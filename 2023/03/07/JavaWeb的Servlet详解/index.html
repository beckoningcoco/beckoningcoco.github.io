JavaWeb的Servlet详解

发表于2023-03-07|更新于2023-03-09|博文
|阅读量:

简介

  • 什么是servlert？

    • servlet是运行在服务器端的一个小程序。

    • servlet是开发动态Web资源的一项技术，在整个开发架构中主要承担控制器（Controller）的作用，刚开始本章主要学习了解Servlet的基础概念及使用规范。

    • Servlet是Server Applet的简称，翻译过来就是服务程序。是由sun公司提供的一门用于开发动态Web资源的技术。简单的讲，Servlet就是运行在服务器上的一个小程序，用来处理服务器接收到的请求。

      比如一般的网页程序，是由我们通过浏览器来访问实现的，在这个过程中，我们的浏览器发送访问请求，服务器接收到请求，并对浏览器的请求做出相应的处理，这就是我们熟悉的B/S模型（浏览器—服务器模型）。而Servlet就是对请求做出处理的组件，运行于支持java的应用服务器中。

      Servlet从狭义上说白了就是个接口，从广义上说就是实现了这个接口的所有的子类。

      附加：我们打开项目目录下的JSP文件查看其源码，可以发现JSP底层也是实现了Servlet接口的，所以JSP它本质上就是一个Servlet。

  • Servlet在其中（MVC架构模式）承担的作用是Controller（控制器）,起到对数据进行操作的作用。

Servlet的开发流程

(一）、实现Servlet接口或者继承HttpServlet类

​ 注：继承HttpServlet类更简洁更常用。

（二）、重写HttpServlet类的doGet()和doPost()方法

（三）、配置web.xml文件或者使用注解对servlet进行配置

  • 首先使用maven创建一个 web项目，并在pom中导入servlet的相关依赖。

  • 使用web.xml配置servlet访问路径（不常用的方式）

  • 使用注解配置servlet访问路径(常用)

注意：配置访问路径，注解和web.xml两者二选一即可，避免混淆！

Servlet 执行原理

  1. 当服务器接收到客户端浏览器的请求以后，会解析url路径，获取访问的Servlet的资源路径。
  2. 查找 Web.xml 文件，看看是否有对应的 标签体内容。
  3. 如果有，再找到对应的全类名。
  4. tomcat将其字节码文件加载进内存，并创建其对象。
  5. 调用对应方法。

Servlet生命周期

  • init方法在第一次访问页面时进行，只进行一次。
  • service方法，每次访问对应路径时都会执行一次。
  • destory方法，服务器关闭时，执行一次。

Servlet生命周期详解

  1. 被创建：init方法执行，只会执行一次

    • 在第一次访问时执行，但是可以通过配置改为在服务器启动时创建。(在 web.xml的servlet标签中配置)

    • init只初始化一次，意味着，Servlet在内存中只存在一个对象，是单例的。

  2. 提供服务：执行service方法，能执行多次

  3. 被销毁：执行destory方法，只执行一次。

    • 只有在服务器正常关闭时，才会执行这个方法。

Servlet体系结构

  1. GenericServlet 实现了Servlet接口，并且对Servlet中的除了service方法之外的方法做了空实现，service继续作为抽象方法保留，这样的好处是，我们自己写的Servlet继承了GenericServlet之后，我们不需要麻烦地重写5个方法，只需要重写service方法即可。
  2. HttpServlet继承了GenericServlet并进一步完善，它在Service方法中添加了判断请求方式的逻辑，当浏览器提交的请求为get时，它会去执行doGet（）方法，post同理，这样，我们自己写的Servlet继承HttpServlet之后，只需要根据请求的方式，重写doGet()或者doPost（）…..之类的方法就可以了。
  3. 在HttpServlet中，大家可以看它的原代码，有2个service方法，一个是保护的（主要看里面的实现），一个是公有的，公有的service接收到请求时，会抛给保护的那个service，它在这只是起一个中转的作用。当前端有请求过来时，会默认找到这2个service方法，具体先找哪个我不清楚，反正最后请求都会在保护的那个service那里，在那个service中，会根据前端是哪种方式的请求，再抛给相应的处理方法，如：get则交给doGet处理。所以，在以继承HttpServlet的方式创建一个servlet时，你只要重写相应的doGet或者doPost等处理方法就行了，然后再在里面写你的代码就行了。

Servlet相关配置

  • 一个Servlet可以有多个访问路径 ：@WebServlet（{“/demo1”，“/demo11”，“/demo111”}）

Servlet中的 request 和 response

  • request 对象和 response对象都是由tomcat服务器创建的，我们来使用它们。
  • request是用来来获取请求消息，response是用来设置响应消息的。

request对象的体系结构

request 对象功能

  • 获取 请求行数据

    • GET /ServletDemo1/web/demo1 HTTP/1.1 这是一个请求行数据。

    • 获取请求方式：String getMethod( ) //GET

    • 获取虚拟目录（tomcat中设置的项目名）：String getContextPath( ) //ServletDemo1

    • 获取Servlet路径：String getServletPath() // /web/demo1

    • 获取get请求方式的拼接参数 （例如 ？name=zhangsan）String getQueryString()

    • 获取请求URI : String getRequestURI ( ) //ServletDemo1/web/demo1

    • 获取请求URL：String getRequestURL（） // http://localhost:8080/ServletDemo1/web/demo1

    • 获取Http协议及版本：String getProtocol（） //HTTP/1.1

    • 获取客户机的IP地址：String getRemoteAddr（）

  • 获取请求头数据

    • String getHeader（String name） 通过请求头的名称获取请求头的值
    • Enumeration getHeaderNames() 获取所有的请求头名称。

  • 获取请求体数据

    • 请求体：只有POST请求有请求体，在请求体中封装了请求参数。

    • 步骤：

      • 获取流对象： BufferrdReader getReader（）获取字符流
      • 从流对象中获取数据

​

  • 其他（更加好用）

    • 获取请求参数通用方式：

      • String getParameter（String name）：根据参数名称获取参数值， username=zs&password=123

      • String[ ] getParameterValues(String name) :根据参数名获取参数值数组，hobby=1&hobby=2…..

      • Enumeration getParameterNames（）：获取请求的所有参数名称。

      • Map<String,String[]> getParameterMap( ):获取参数的Map，里面存的键值对。

请求转发

  • 一种在服务器内部进行资源跳转的方式。
  • 步骤：
    • 通过request对象获取请求转发器对象：RequestDispatcher getRequestDispatcher （String path）
    • 使用请求转发器对象进行转发：forward（ request ， response）
    • 特点：
      • 浏览器地址栏路径不会发生改变
      • 只能转发到当前服务器内部的资源中
      • 转发是一次请求（虽然访问了两个Servlet但是一次请求）

共享数据

  • 域对象：一个有作用域范围的对象，可以在域范围内共享对象

  • request域：代表一次请求的范围，一般用于请求转发多个资源中的共享数据

    • 方法：setAttribute（String name ，Object obj）; 存储数据
    • Obj getAttribute（String name）；通过键 获取值
    • removeAttribute（String name）； 通过键移除值

  • 在哪个作用域中设置共享数据，就只能从该作用域中取出数据。

  • 测试 ：请求Servlet05，存一个name=zs，转发到Servlet06，取出name。

获取ServletContext

  • 在servlet类中：super.getServletContext();
  • 通过请求对象来获取：request.getServletContext();
  • 通过session对象来获取：request.getSession().getServletContext();
  • 注意点：无论是哪一种方式，获取的都是同一个ServletContext对象

response对象

  • 封装了所有响应信息并将服务器端资源数据输出到浏览器
  • HttpServletResponse 对象封装了向客户端发送数据、发送响应头，发送响应状态码的 方法。

常用API

页面跳转—重定向

页面转发和重定向的区别？

  1. 请求转发url没有发生变化，//重定向url发生变化

  2. 请求转发跳转发生在服务器内部，重定向跳转发生在浏览器客户端

  3. 请求转发是服务器内部跳转，只能访问当前资源目录里面的资源；

​ 重定向是客户端浏览器发生跳转，所以可以访问本服务器上任意位置资源

  1. 请求转发共享一个request和response，重定向不共享。

  2. 请求转发访问的是同一目录下的资源,所以访问速度快 ;

​ 重定向可以访问本服务器上的任意位置资源,所以访问速度比较慢.

ServletContext对象

  • 概念：代表整个web应用，可以和服务器进行通讯

  • 获取：

    • 通过request获取 request.getServletContext();
    • 通过httpServlet获取this.getServletContext();

  • 功能：

    • 获取MIME类型

      • 格式 ： 大类型/小类型 text/html image/jpeg
      • 获取：String getMimeType（ String file）

    • 域对象：共享数据

    • 获取文件的真实（服务器）路径

      • 方法：getRealPath（String path）
