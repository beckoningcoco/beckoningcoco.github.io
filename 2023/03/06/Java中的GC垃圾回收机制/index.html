Java中的GC垃圾回收机制

发表于2023-03-06|更新于2023-03-06
|阅读量:

初步了解什么是GC （垃圾回收）

  1. 什么是垃圾？

    • 垃圾是指在运行程序中没有任何指针指向的对象，这个对象就是需要被回收的垃圾。

  2. 我们为什么需要 GC ?

    • 如果不进行垃圾回收，内存迟早会被消耗完。
    • 垃圾回收可以清除内存中的记录碎片，碎片整理将堆内存中的碎片移动到堆的一段，以便JVM将整理出新的内存分配给新的对象。

  3. 早期的 GC回收

    • 在早期的C/C++时期，垃圾回收基本是手工进行的，开发人员利用new关键字进行内存申请，利用delete关键字进行内存释放。

    • 这种方式十分灵活的控制内存的释放时间，但是，频繁的申请和释放内存，会给开发人员带来不小的管理负担，一旦忘记回收某段内存空间，极有可能造成内存泄漏，导致垃圾对象永远无法被清除，随着时间的积累，垃圾对象占用的内存越来越多，直到内存溢出，导致应用程序崩溃！

Java垃圾回收机制

  1. 首先，我们关心JVM中的哪些区域用到了GC回收机制？
    • 方法区
    • 堆（重点区域）
  2. 垃圾回收可以对年轻代回收，也可以对老年代回收，甚至是全堆和方法区的回收。
    • 频繁收集 年轻代区
    • 较少收集老年代区
    • 几乎不收集永久代区

垃圾回收相关算法

标记阶段：引用计数算法

  • 所谓垃圾表示阶段，就是判断哪些对象是垃圾，哪些对象还存活的阶段。
  • 什么是引用计数算法？
    • 对每个对象保存一个整形的引用计数器属性，用于记录对象被引用的情况。
    • 对于一个对象A，只要有任何一个对象引用了A，则A的引用计数器+1，当引用失效时，计数器-1。只要对象的引用计数器为0，就代表对象不再有任何引用，可以被回收了。
    • 优点：实现简单，垃圾对象便于辨识；判定效率高，回收没有延迟性。
    • 缺点：
      • 需要单独的字段存储计数器，增加了存储空间的开销
      • 每次赋值都需要更新计数器，伴随着加减法操作，增加了时间开销
      • 最严重的是，无法解决循环引用的情况，这太致命了，导致java没有使用这类算法。

标记阶段：可达性分析（又叫根搜索算法，追踪性垃圾收集）

  • 相对于引用计数算法而言，可达性分析算法不仅同样具备实现简单和执行高效的优点，更重要的是，该算法能够有效的解决引用计数算法中的循环引用的问题，防止内存泄漏的发生。（内存泄漏是指：内存中用没用的，但是无法被回收的对象。）

  • Java，C#,选择的就是 可达性分析算法。

介绍：

  • 可达性分析算法，是以根对象集合（GC Roots 根集合，是一组必须活跃的引用）为起始点，按照从上到下的方式，搜索被根对象集合所连接的目标对象是否可达。

  • 使用可达性分析算法之后，内存中所有的存货对象都会直接或者间接的被根对象连接着，搜索所走过的路径叫做引用链

  • 如果目标没有任何引用链连接着，则以为着目标不可达，目标为垃圾对象。

  • 在可达性分析算法中，只有被引用链连接着的对象，才能被认为是存活对象。

  • GC Roots到底是什么？
    • 简单理解：引用了堆空间或方法区中的对象，但是自己并不在堆内存之中，就是一个Root。
    • 例如，栈内的变量和指针。
