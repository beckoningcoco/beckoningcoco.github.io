VUE 2.0~3.0

1. vue 简介

1.1 vue 特性

  • 数据驱动视图 ： vue 会监听 数据的变化，从而自动重新渲染页面

    好处：数据变化，页面自动渲染

    坏处：单向的数据绑定

  • 双向数据绑定：填写表单时，可以辅助开发者不操作dom的情况下，自动把用户填写的数据同步到数据源中

    好处 ： 开发者不需要手动操作 DOM元素来获取表单元素最新值

1.2 MVVM

  • MVVM是 vue实现数据驱动视图和双向数据绑定的核心原理，MVVM是指 Model ， view ， ViewModel，每个html页面都被拆分成这三个部分。
  • Mode ：当前页面渲染所需要的数据源。
  • View ： 当前页面所渲染的Dom结构
  • ViewModel：表示 vue 实例，他是 MVVM的核心。

1.21 MVVM的工作原理

  • ViewModel 作为MVVM的核心，是它把当前页面的数据源 和页面的数据结构匹配起来。

  • 当数据源放生改变时，ViewModel 会监听到，VM会根据最新的数据源自动更新页面结构

  • 当表单元素发生改变时，VM会监听到，把变化后的最新值同步到数据源Model中

    1. 内容渲染指令
    2. 属性绑定指令

3.6 事件修饰符

  • 在事件处理函数中调用 event.preventDefault( ) 或者 event.stopPropagation( )是非常常见的需求。因此vue也为我们提供了事件修饰符的概念。
    1. .prevent:阻止默认行为 （例如阻止a链接的跳转，阻止表单的提交）
    2. .stop：阻止事件冒泡
    3. .capture ：以捕获模式触发当前事件的处理函数
    4. .once：绑定的事件只触发一次
    5. .self ：只有在 event.target 是当前元素自身时，才会触发事件处理函数

3.81 v-model 的 指令修饰符

  • 为了方便对用户输入的内容进行处理，有三个修饰符。
    1. .number : 自动将用户输入的内容转换为数值类型
    2. .trim : 自动过滤用户输入的 首尾 空白字符
    3. .lazy ：在”change” 时 而非 “input” 时更新

3.9 条件渲染指令 v-if v-show

  • v-if ：为true时，显示标签，false时，不显示标签
  • v-show：效果与 v-if 相同
  • 二者区别：
    • v-show原理 是 ：动态为元素添加或者移除 display：none的样式，来实现元素的显示或者隐藏
    • v-if原理是，每次动态创建和移除元素，实现元素显示和隐藏。
    • 需要频繁切换显示状态，用 v-show性能更好，不频繁的用 v-if。实际上电脑性能这么好，二者区别不大。
