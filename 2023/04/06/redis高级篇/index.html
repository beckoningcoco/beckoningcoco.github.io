单点Redis存在的问题

  • 数据丢失问题：数据存在内存中，一断电，咔全没了~
  • 并发能力问题: 现在的并发还算不错，可到双十一那种，数百万的访问量，单点redis就不够用了！
  • 存储能力问题: 内存存储，能存多少东西~
  • 故障恢复问题：单个redis服务器，要是一断电，咔，服务都不能用了~

Redis持久化

RDB持久化

  1. 什么是RDB？

    RDB全称：Redis Database Backup file （redis数据备份文件），也被叫做redis数据快照。简单来说就是把内存中的所有数据都记录到磁盘之中。当redis实例故障之后，能从磁盘读取文件，恢复数据。

    快照文件成为RDB文件，默认保存在当前运行目录。

    我们主动使redis停机时，会自动做一次备份RDB。（再次启动时，还是可以拿到之前的数据的。）

多级缓存–亿级流量的缓存方案

传统缓存的问题

传统的缓存策略一般是请求到达Tomcat之后，先查询Redis，如果未命中则查询数据库，存在以下问题。

  • 请求要经过Tomcat处理，Tomcat的性能成为整个系统的平静
  • Redis缓存失效（过期）时，会直接查询数据库，产生冲击。

​

多级缓存方案

多级缓存就是充分利用请求处理的每个环节，分别添加缓存，减轻Tomcat压力，提升服务性能:
